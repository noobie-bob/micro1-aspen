A teammate on the platform pinged me — they noticed some strange behavior while using our new FastMCP ProjHub tools through their local client. They mentioned that some administrative tools don't seem to enforce proper authorization checks. They also noticed that some legacy tools are still accessible and expose sensitive system secrets.

They kept poking around and found multiple injection vulnerabilities across different tools. In some tools, they were able to execute arbitrary system commands or inject malicious code snippets. In others, they found they could fetch internal network resources. They even found a way to trick the underlying LLM into regurgitating internal system context using malicious input. To top it all off, when an error occurs, the server spits out overly verbose error details.

We don't have adequate security tests covering these authorization flaws, injection vulnerabilities, and data exposure issues. Your job is to write the test suite that catches all of them.

**Write tests only. Do not modify any file under `mcp_server/`.** A separate engineer will use your tests to validate their fix; for them to trust your tests, the suite must (1) catch all the security issues the reporter is exploiting, and (2) catch a fix that over-blocks legitimate flows. Put your tests under `tests/exfiltration/`. The smoke tests in `tests/test_smoke.py` show the legitimate flows already in use — your suite should pin those down with anti-overblock assertions so they cannot regress.

Submit only your test files as a unified diff via `submit_answer` (applies clean from `/repo`).
