The seeded vulnerabilities in this FastMCP server span multiple OWASP classes
including specific security gaps (Command Injection, SSRF, Prompt Injection, etc.)
all contextualized within a MCP-like domain:

1. Tools endpoints suffer from OS command injection (ping_team_server) and unsafe eval() (calculate_budget)
2. Resources endpoints blindly fetch external URLs (SSRF) and load untrusted plugins (load_custom_formatter)
3. Prompts endpoints lack sanitization, enabling direct and indirect prompt injection (summarize_task)
4. The authentication wrapper has a logic bypass (bypass- prefix)
5. Legacy tools expose hardcoded credentials and exceptions leak tracebacks

Test-authoring is the right shape because the threat surface is spread across multiple FastMCP tools,
requiring careful testing to patch without breaking valid domain tools.

Current rubric design:
- 9 critical items (×4): code injection, credentials multi-caller, SSRF nonfetch, module loader controls,
  eval dual-contract, multi-variant SSRF, legacy diagnostic multi-caller, ping multi-payload sweep,
  cross-team search isolation
- 5 major items (×3): command injection, module path rejection, regression guards (×2), delete-verify chain
