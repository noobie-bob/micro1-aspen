They also did some deeper poking around and found a severe database injection vulnerability, discovered some leftover administrative debugging endpoints that expose sensitive data, and noticed that error handling is dangerously verbose.

We need a comprehensive security test suite covering all of these access control, data exposure, and injection vulnerabilities. Your job is to write the test suite that catches all of them.

**Write tests only. Do not modify any file under `projhub/`.** A separate engineer will use your tests to validate their fix; for them to trust your tests, the suite must (1) catch the cross-team data leaks and other security issues, and (2) catch a fix that over-blocks legitimate same-team or admin flows.

Put your tests under `tests/exfiltration/`. `tests/test_smoke.py` shows the legitimate participant + admin flows already in use — your suite should pin those down with anti-overblock assertions so they cannot regress.
