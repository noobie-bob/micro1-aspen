The seeded vulnerabilities in this project-management API span multiple OWASP
classes, all centered on a Flask service managing teams, projects, tasks,
comments, and attachments:

1. Cross-team data isolation failure (BOLA/IDOR): most read/write endpoints
   beyond creation lack team-membership checks.
2. Missing field redaction: admin-only fields (internal_priority, reviewer_notes,
   internal_metrics, admin_config, budget_allocation) are never stripped.
3. SQL injection: search endpoints use raw string interpolation into SQLite queries.
4. OS command injection: report generation passes user input to shell commands.
5. Mass assignment: PATCH endpoints accept admin-only fields from non-admin callers.
6. Sensitive data exposure: debug endpoints leak plaintext API tokens, environment
   variables, and system configuration without authentication.
7. Information exposure: error handler returns full Python stack traces and framework
   version to callers.
8. SSRF: webhook endpoint fetches user-supplied URLs server-side without validation.
9. Weak cryptography: share-link token derived from MD5 of project ID.
10. Path traversal: attachment download accepts path segments without sanitization.
11. Admin endpoint access: admin endpoints use require_auth instead of require_admin.
12. Deprecated APIs: legacy v1 endpoints serve data without any authentication.

Test-authoring is the right shape because the threat surface spans 25+ endpoints
across 9 route modules. The rubric decomposes along structural axes covering all
vulnerability classes above.
